# Kufar rental monitor

Мониторинг новых объявлений об аренде квартир на Kufar (re.kufar.by) с
уведомлениями в Telegram. Cloudflare Worker, Cloudflare Cron Trigger
опрашивает раз в 5 минут, состояние (просмотренные id) — в Workers KV.

## Деплой

1. Установи Wrangler и залогинься:

   ```bash
   npm install -g wrangler
   wrangler login
   ```

2. Создай KV namespace и подставь его id в `wrangler.toml`
   (`kv_namespaces[0].id`):

   ```bash
   wrangler kv namespace create KUFAR_KV
   ```

3. Задай секреты (не хранятся в репозитории):

   ```bash
   wrangler secret put TELEGRAM_BOT_TOKEN
   wrangler secret put TELEGRAM_CHAT_ID
   ```

   `TELEGRAM_CHAT_ID` — id чата/канала, куда бот должен писать (бот должен
   быть туда добавлен и иметь право отправлять сообщения).

4. При необходимости поменяй `SEARCH_URL` в `wrangler.toml` (город,
   валюта, размер страницы и т.д.).

5. Задеплой:

   ```bash
   wrangler deploy
   ```

## Проверка после деплоя (обязательно)

Код не проверялся на реальном HTML-ответе Kufar. После деплоя:

1. Открой URL воркера в браузере или дёрни curl-ом — это вызовет
   `fetch`-обработчик и вернёт текстовый отчёт: HTTP-статус, размер HTML,
   сколько объявлений найдено, сколько из них новых.

2. **Первый запуск особый**: если ключ `seen_ids` в KV ещё не существует,
   воркер только инициализирует базу найденными id и НЕ шлёт уведомления
   (иначе при первом запуске в Telegram улетит сразу ~30 сообщений).
   Все последующие запуски будут присылать только реально новые id.

3. Если в отчёте `Найдено объявлений: 0` — воркер сам вернёт фрагмент
   HTML (первые 2000 символов), в отчёте. По нему нужно посмотреть,
   как реально выглядит разметка ссылок на объявления, и поправить
   регулярку `AD_LINK_RE` в `worker.js` под неё.

4. Если HTTP-статус в отчёте не 200 (403/429/редирект и т.п.) — это
   похоже на блокировку запросов с IP Cloudflare Workers. В таком случае:
   - попробовать другие заголовки (полный набор как у настоящего Chrome:
     `sec-ch-ua`, `sec-fetch-*` и т.д.);
   - либо обсудить альтернативный способ получения данных (например,
     прокси/сторонний рендер), так как обход WAF выходит за рамки этого
     воркера.

5. Убедиться, что Telegram-бот реально присылает сообщения: можно
   временно удалить ключ `seen_ids` из KV (`wrangler kv key delete
   seen_ids --binding KUFAR_KV`), а затем ещё раз удалить его перед
   финальным прогоном, либо просто дождаться следующего цикла — если
   на сайте появится новое объявление, придёт сообщение со ссылкой.

## Логи

```bash
wrangler tail
```

Полезно во время первых ручных прогонов, чтобы увидеть ошибки сети/KV
в реальном времени.
